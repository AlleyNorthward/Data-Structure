# 简介
这部分呢, 其实跟外界完全剥离开来的, 可以灵活地将整个文件拿走, 在当前目录下, 也能够正常地编译执行的. 结果就是, 存在重复文件(`data_structurelib data_structure_cxxlib`在外面就有).  
为什么选择这么做呢? 因为这只是`c/cxx->python`的一部分. 后续, 我可能会专门弄一个`python`相关的编译库, 专门存储`c/cxx->python`的, 那么, 这个数据结构的必然是少不了的. 所以, 如果此时编译链接到外部的话, 影响就是无法完全剖离, `CMakeLists.txt`之间或多或少地有牵连, 修改是必然的. 为了减少后面移植的困难, 所以直接将上面两个文件放在当前目录下面, 这样就不用来回查找/修改了, 直接移植就可以了.  
缺点也是有的, 全部是静态库, 文件会稍大些. 是否需要换成动态库呢? 以后再说吧. 毕竟目前来看, 文件并不是很大, 此时也能解决大部分问题.
